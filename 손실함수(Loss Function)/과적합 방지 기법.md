# 과적합 방지 기법

### 1. 라벨 스무딩
> [!NOTE]
> 만약 사람, 자동차, 음식를 분류한다고 가정할 때<br>
> 사진을 보고 순서대로 `[0.7, 0.2, 0.1]`이라고 예측한 경우<br>
> 사람일 확률은 70%, 자동차는 20%, 음식은 10%이다.<br>
> 그런데 `[1.0, 0.0, 0.0]`인 경우 지나치게 확신한 상태가 돼 일반화 능력을 저하시킨다.<br>
> 즉, 하나를 보고 확신을 해버리면 문제가 조금이라도 바껴도 틀리게 돼 융통성이 없어진다.<br><br>
> 이렇듯 너무 과신할 경우 일반화 능력이 떨어지므로 `라벨 스무딩`기법을 사용한다.<br>
> 정답 레이블이 `[1.0, 0.0, 0.0]`인 경우 `[0.933, 0.033, 0.033]`같이<br>
> 정답 레이블을 부드럽게 만들어 일반화 성능을 향상시키고 과적합 방지, 예측 확률의 신뢰도를 증가시킨다.<br><br>
**수식:** `y_smooth = y × (1 - α) + α / K`
- α: 스무딩 파라미터 (α 권장값: 0.1 (ImageNet 표준))<br>
- K: 클래스 개수<br>
> 예시<br>
> `클래스 0   : 1×0.9 + 0.1/3 = 0.933`<br>
> `클래스 1, 2: 0×0.9 + 0.1/3 = 0.033`<br>
