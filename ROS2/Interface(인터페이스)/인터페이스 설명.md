# 인터페이스 개념
> 인터페이스: 노드끼리 주고받는 데이터의 형식 정의(보통 인터페이스 패키지를 따로 만들어서 관리)  
> 웬만하면 기존에 정의된 표준 인터페이스를 사용하고 원하는 게 없는 경우에만 커스텀 인터페이스 생성.  
> 왜냐하면 표준 인터페이스는 표준 도구 및 드라이버를 사용할 때 편의성이 좋다.  

## 인터페이스 세 종류
### 1. 메시지(.msg): 토픽으로 계속 흘려보냄(센서, 상태)
```msg
string robot_id
float64 battery
bool is_charging
```
### 2. 서비스(.srv): 요청 -> 응답(한 번 묻고 답)
```srv
int64 a
int64 b
---
int64 sum
```
### 3. 액션(.action): 오래 걸리는 일 + 진행상황

## 인터페이스 패키지 생성 방법
> 빌드타입(build type): `ament_cmake`  
> 예시 코드: `ros2 pkg create --build-type ament_cmake my_interfaces`  
> 주의할 점: `/src 폴더`에서 작업하는 걸 깜빡하지 말 것!  

## 위에서 소개한 인터페이스 세 종류 만드는 법
> 해당 패키지 내에 원하는 타입에 맞는 폴더(`msg`, `src` 등)을 만들고 파일명은 `.msg`처럼 폴더에 맞게 작성하면 된다.  

## 의존성 해결
> 폴더를 만들고 파일까지 작성했다면 아래 작업들을 실행해줘야 환경 세팅까지 완료된다.  
1. **CMakeLists.txt**: `rosidl_generate_interfaces` 내부에 작성한 파일명을 작성한다.
```txt
rosidl_generate_interfaces(${PROJECT_NAME}
  "msg/RobotStatus.msg"
  "srv/AddTwoInts.srv"
)
```

2. **package.xml**: 다른 패키지 형식을 중첩했다면 `<depend>` 태그 안에 해당 형식을 기재해야 한다.
```xml
<depend>
    geometry_msgs
<depend>
```

3. 위의 과정을 다 했다면 `ros2_ws`에서 `colcon build`와 `source install/setup.bash`를 하면 끝!!
