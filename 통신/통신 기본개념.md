# 통신

## 통신 프로토콜

### 1. TCP (Transmission Control Protocol)
> [!NOTE]
> TCP는 연결(Connection)을 먼저 설정한 뒤 데이터를 주고받는 연결 지향(Connection-Oriented) 프로토콜이다.<br>
> 데이터가 순서대로, 중복 없이, 손실 없이 전달되도록 확인(ACK)과 재전송 기능을 제공하여 신뢰성이 높다.<br>

- 특징<br>
> - 송신자와 수신자가 먼저 연결을 설정한 후 통신한다.
> - 데이터의 순서를 보장하며 손실된 데이터는 재전송한다.
> - 오류 검출 및 흐름 제어, 혼잡 제어 기능을 제공한다.
> - 신뢰성은 높지만 확인 및 재전송 과정 때문에 속도는 UDP보다 느리다.

- 통신 과정<br>

```text
Client                  Server
  | ---- SYN ---------> |
  | <--- SYN + ACK ---- |
  | ---- ACK ---------> |
  | ===== 데이터 =======> |
  | <==== ACK ========= |

연결 설정 → 데이터 전송 → 연결 종료
```

- 사용 예시<br>
> 웹 브라우징(HTTP/HTTPS), 이메일(SMTP), 파일 전송(FTP), 메신저, 금융 거래 등<br>
> 데이터의 정확한 전달이 중요한 서비스에서 사용된다.

---

### 2. UDP (User Datagram Protocol)
> [!NOTE]
> UDP는 연결을 설정하지 않고 데이터를 전송하는 비연결형(Connectionless) 프로토콜이다.<br>
> 데이터의 전달 여부나 순서를 확인하지 않기 때문에 신뢰성은 낮지만 전송 속도가 빠르고 지연이 적다.<br>

- 특징<br>
> - 연결 과정 없이 바로 데이터를 전송한다.
> - 데이터 손실이나 순서 변경이 발생할 수 있다.
> - 재전송이나 오류 복구 기능이 없어 오버헤드가 적다.
> - 실시간성이 중요한 서비스에 적합하다.

- 통신 과정<br>

```text
Client                  Server
  | ---- 데이터 ------> |
  | ---- 데이터 ------> |
  | ---- 데이터 ------> |

연결 과정 없이 즉시 데이터 전송
```

- 사용 예시<br>
> 실시간 영상 스트리밍, 인터넷 방송, 음성 통화(VoIP), 온라인 게임, DNS 등<br>
> 약간의 데이터 손실보다 빠른 전송이 중요한 서비스에서 사용된다.

---

### TCP vs UDP 비교

| 항목 | TCP | UDP |
|------|-----|-----|
| 연결 방식 | 연결 지향 | 비연결형 |
| 신뢰성 | 높음 | 낮음 |
| 전송 속도 | 느림 | 빠름 |
| 순서 보장 | O | X |
| 재전송 | O | X |
| 대표 사용 | HTTP, HTTPS, FTP | DNS, VoIP, 게임 |

---

## 파일 형태

### 1. JSON (JavaScript Object Notation)
> [!NOTE]
> JSON은 사람이 읽기 쉽고 컴퓨터가 처리하기 쉬운 경량 데이터 교환 형식이다.<br>
> 키(Key)-값(Value) 형태로 데이터를 표현하며 다양한 프로그래밍 언어에서 쉽게 사용할 수 있다.<br>

- 특징<br>
> - 중괄호 `{}`와 대괄호 `[]`를 사용하여 데이터를 표현한다.
> - Key와 Value 형태로 데이터를 저장한다.
> - 데이터 크기가 작아 전송 속도가 빠르다.
> - 사람이 읽기 쉽고 대부분의 언어에서 지원된다.

- 예시<br>

```json
{
  "name": "Robot",
  "speed": 1.5,
  "enabled": true
}
```

- 사용 예시<br>
> REST API 데이터 교환, 웹 서버와 클라이언트 간 통신, 설정 파일(package.json), NoSQL 데이터 저장 등
> 웹 개발과 다양한 네트워크 통신에서 가장 많이 사용되는 데이터 형식이다.

---

### 2. XML (eXtensible Markup Language)
> [!NOTE]
> XML은 태그(Tag)를 이용하여 데이터를 구조적으로 표현하는 마크업 언어이다.<br>
> 데이터의 구조를 명확하게 표현할 수 있으며 확장성이 뛰어나 다양한 시스템 간 데이터 교환에 사용된다.<br>

- 특징<br>
> - 사용자가 직접 태그를 정의할 수 있다.
> - 계층적인 구조로 데이터를 표현한다.
> - JSON보다 데이터 크기가 커지는 경향이 있다.
> - 사람이 읽기 쉽지만 태그가 많아 다소 복잡하다.

- 예시<br>

```xml
<robot>
    <name>Robot</name>
    <speed>1.5</speed>
    <enabled>true</enabled>
</robot>
```

- 사용 예시<br>
> 웹 서비스(SOAP), 설정 파일, 문서 저장 형식, RSS 피드, 시스템 간 데이터 교환 등<br>
> 구조가 복잡한 데이터를 표현하거나 호환성이 중요한 환경에서 사용된다.

---

### 3. YAML (YAML Ain't Markup Language)
> [!NOTE]
> YAML은 사람이 읽기 쉽도록 설계된 데이터 직렬화 형식이다.<br>
> 들여쓰기를 이용해 계층 구조를 표현하며 설정(Configuration) 파일에 많이 사용된다.<br>

- 특징<br>
> - 들여쓰기로 계층 구조를 표현한다.
> - 중괄호나 태그를 거의 사용하지 않아 가독성이 높다.
> - JSON보다 사람이 작성하고 수정하기 쉽다.
> - 공백과 들여쓰기가 문법에서 매우 중요하다.

- 예시<br>

```yaml
name: Robot
speed: 1.5
enabled: true
```

- ROS 2 설정 파일 예시<br>

```yaml
turtle:
  ros__parameters:
    linear_speed: 2.0
    angular_speed: 1.0
```

- 사용 예시<br>
> Docker Compose, Kubernetes, GitHub Actions, Ansible, ROS 2 설정 파일 등<br>
> 서버 및 개발 환경의 설정 파일에서 가장 많이 사용되는 데이터 형식이다.

---

### JSON vs XML vs YAML 비교

| 항목 | JSON | XML | YAML |
|------|------|-----|-------|
| 가독성 | 높음 | 보통 | 매우 높음 |
| 데이터 크기 | 작음 | 큼 | 가장 작음 |
| 표현 방식 | Key-Value | 태그(Tag) | 들여쓰기 |
| 사람이 작성하기 | 쉬움 | 보통 | 가장 쉬움 |
| 대표 사용 | REST API | 문서, SOAP | 설정 파일 |

> [!TIP]
> JSON, XML, YAML은 모두 같은 데이터를 표현할 수 있는 형식이다.<br>
> 차이점은 데이터를 표현하는 문법이며 목적과 사용 환경에 따라 적합한 형식을 선택한다.
